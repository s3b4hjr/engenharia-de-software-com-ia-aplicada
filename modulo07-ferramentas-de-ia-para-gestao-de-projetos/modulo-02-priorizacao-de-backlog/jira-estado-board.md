## Estado esperado: Backlog plano, Sprint 1 não iniciado

Neste módulo o board serve como referência visual das User Stories que serão priorizadas com IA. O foco é no backlog — os itens ainda não têm ordem de execução definida.

### Checklist de configuração

- [ ] Backlog importado (mesmo estado do Módulo 1.2)
- [ ] Sprint 1 visível no backlog, mas **não iniciado**
- [ ] Todas as histórias com status **"A fazer"**
- [ ] Nenhum sprint ativo
- [ ] Story points visíveis nas histórias (se não aparecerem, ative em Project settings → Features → Estimation)

### Como verificar

Abra o Backlog. Nenhuma sprint deve aparecer como "Ativo". As histórias devem estar listadas sem ordem de prioridade definida — é exatamente esse estado que a ferramenta de IA vai nos ajudar a resolver neste módulo.

### Ponto de partida igual ao Módulo 1.2

Se você já configurou o board para o Módulo 1.2 e não fez alterações desde então, o board já está no estado correto para este módulo.
